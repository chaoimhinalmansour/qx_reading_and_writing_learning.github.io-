# IELTS Chunk Daily

这是一个本地静态原型，用来把“词 + 词块”知识库转成每日雅思 Academic 阅读与写作练习题卡。

## 已实现

- 页面拆成两个板块：阅读练习、写作练习。
- 两个板块共用动作链：认识单词、语境理解后提取表达、判断词块含义和作用、应用练习。
- 顶部提供开发者模式开关；打开后直接解锁所有步骤，方便调试，不代表学生端真实流程。
- 认词流程：每次只出现一个来源词，句子直接显示；学生必须先点按钮听整句，听完后才开放四选一中文义判断。
- 认词反馈：第一次选错会允许重新选一次；第二次仍错则揭示正确含义，然后进入下一词。
- 内置词块知识库字段：word、chunk、中文释义、来源词释义、来源词轻例句、正确例句、主题标签、功能标签、错误样例、可替换表达。
- 学生端不要求先掌握功能术语；功能标签作为后台出题和反馈字段使用。
- 初始状态不显示完整目标词块；学生先完成认词，再进入对应板块的表达提取。
- 阅读板块不再直接展示来源词；学生先根据“信息任务”选择信息所在句，再判断句意。完成这两步后，系统显示目标短语，学生只需照着写一遍。
- 检查后才揭示“目标短语 -> 可套用形式 -> 替代表达 -> 写作提示”。
- Part 2 不做开放式短语提取或模板归纳，避免软件误判学生五花八门的回答；自由调用放到后面的应用练习。
- 写作板块也不直接展示来源词；学生先判断表达意图，再看到目标短语并照着写一遍。
- 中文含义判断采用轻量宽判，允许“很难”“不容易”“导致/造成”等接近表达通过。
- 关系判断只围绕知识库词块，不再用未入库的普通连接词作为题目主角。
- 轻反馈规则：每个部分给一个关键反馈点，不做完整雅思 band 批改。
- 迁移判断：如果学习者完成识别但没有在改写或总结中调用目标词块，会显示“未完成迁移”。
- 三张示例题卡：词汇准确使用、Task 1 趋势表达、Task 2 让步论证。

## 使用方式

直接打开 `index.html` 即可使用，不需要安装依赖。

## 后续接入真实知识库

当前知识库写在 `app.js` 的 `KNOWLEDGE_BASE` 和 `DAILY_CARDS` 中。接入真实数据时，保留这些字段即可：

- `word`
- `chunk`
- `zh`
- `sourceZh`
- `sourceExample`
- `example`
- `themeTags`
- `functionTags`，后台字段，不直接暴露给学生
- `errorExample`
- `alternatives`

每张题卡还可以配置：

- `sourceCards`：包含 `word`、`sentence`、`answer`、`choices`
- `reading` / `writing`：分别配置 `retrieval`、`judge`、`application`
